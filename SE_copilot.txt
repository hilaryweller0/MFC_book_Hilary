The spectral element method (SEM) is a high‑order numerical technique for solving partial differential equations (PDEs) that combines ideas from:

Spectral methods (very high accuracy using global polynomials), and
Finite element methods (FEM) (geometric flexibility via domain decomposition).
It is widely used in computational fluid dynamics, wave propagation, and geophysical flows.

Core idea
SEM solves a PDE by:

Decomposing the domain into elements (as in FEM),
Approximating the solution within each element using high‑degree polynomials (as in spectral methods),
Enforcing continuity between elements in a weak (variational) sense.
So instead of:

FEM → many elements, low‑order polynomials
Spectral → one (or few) elements, very high‑order polynomials
SEM uses:

many elements × high‑order polynomials

Relation to other methods
Compared to standard FEM
FEM: low‑order basis (linear, quadratic)
SEM: high‑order basis per element
SEM typically achieves higher accuracy per degree of freedom
Compared to spectral methods
Spectral methods use global basis functions → poor for complex geometries
SEM retains geometric flexibility via elements
Compared to discontinuous Galerkin (DG)
SEM is usually continuous Galerkin
DG spectral element methods also exist and are popular for hyperbolic PDEs
Typical PDEs solved with SEM
Incompressible and compressible Navier–Stokes equations
Wave equations
Advection–diffusion problems
Geophysical and climate models (e.g. atmosphere and ocean dynamics)
A well‑known SEM code is Nek5000 / NekRS.

Key advantages
Very high accuracy for smooth solutions
Efficient on modern HPC architectures
Natural fit for tensor‑product elements
Diagonal (or easily inverted) mass matrices
Limitations
Less effective for non‑smooth solutions (shocks, discontinuities)
More complex implementation than low‑order FEM
Mesh generation still matters
One‑sentence summary
The spectral element method is a high‑order finite element technique that uses spectral‑accuracy polynomial approximations within elements to efficiently solve PDEs on complex geometries.
